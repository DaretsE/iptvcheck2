# 📺 IPTV Analyzer Dashboard

Интерактивная панель, которая проверяет курируемый реестр из **25 публичных
IPTV-плейлистов** на доступность, считает каналы, подбирает EPG (телепрограмму),
строит общий рейтинг и **отдельную таблицу топа русскоязычных листов**.

Работает локально (`streamlit run app.py`) и бесплатно разворачивается на
**Streamlit Community Cloud**. Дополнительно есть headless-режим для
**GitHub Actions**, который ежедневно обновляет `ranking.json` и `RANKING.md`.

---

## Почему предыдущие попытки не работали

- Половина ссылок-источников были выдуманы или вели на неверные пути
  (например, `…/iptv-org/iptv/master/index.m3u` отдаёт **404** — индексы
  iptv-org лежат на `iptv-org.github.io`, а не в raw-репозитории).
- В коде были синтаксические ошибки в регулярных выражениях — приложение
  вообще не запускалось.
- Скрейпинг Telegram через `t.me/s/…` ненадёжен и часто пуст.

Здесь источники — **реально живые и поддерживаемые** проекты (iptv-org,
Free-TV/IPTV и др.), проверенные вручную. Они сами агрегируют каналы со
всего мира, поэтому «поиск везде» обеспечивается широтой реестра.

## Структура

```
iptv-dashboard/
├── app.py                      # интерфейс Streamlit
├── core.py                     # сеть, парсинг, рейтинг (тестируется отдельно)
├── generate.py                 # headless-режим (для Actions/cron)
├── requirements.txt
├── .streamlit/config.toml      # тёмная тема
└── .github/workflows/update.yml
```

## Запуск локально

```bash
pip install -r requirements.txt
streamlit run app.py
```

Открыть в браузере `http://localhost:8501`, нажать **«Найти и обновить базу»**.

## Деплой на Streamlit Community Cloud (бесплатно, ~2 минуты)

1. Создайте репозиторий на GitHub и загрузите туда все файлы.
2. Зайдите на <https://share.streamlit.io>, войдите через GitHub.
3. **New app** → выберите репозиторий, ветку и `app.py` → **Deploy**.

## Headless / GitHub Actions

```bash
python generate.py            # без проверки потоков
python generate.py --probe    # с выборочной проверкой потоков
```

Workflow `update.yml` запускается вручную (вкладка **Actions**) и по
расписанию, обновляя `ranking.json` и `RANKING.md` прямо в репозитории.

## Важные оговорки

- **Гео-блокировки.** Серверы Streamlit Cloud и раннеры GitHub находятся в
  США/ЕС. Часть страновых потоков может не отвечать оттуда, хотя у вас
  локально они работают. Поэтому в рейтинге доступность самого листа и число
  каналов весомее, чем выборочный пинг потоков. Для честной проверки
  потоков запускайте приложение **локально** (или включите режим пробы).
- Это инструмент для агрегирования **открытых, бесплатно распространяемых**
  плейлистов. Не используйте его для платных/пиратских потоков.
- Для просмотра подойдут VLC, Kodi (PVR IPTV Simple Client), TiviMate,
  IPTV Smarters.

## Как считается рейтинг

Прозрачная формула (0–100): доступность листа (40) + число каналов по
лог-шкале (до 30) + доступность потоков по выборке (до 25) + бонус за
быстрый отклик (5). См. `compute_score` в `core.py`.

## Как определяются русскоязычные листы

Лист попадает в русскоязычный топ, если он либо помечен в реестре флагом
`ru` (страны СНГ с русскоязычными каналами), либо доля каналов с кириллицей
в названиях ≥ 40 % (`RU_RATIO_THRESHOLD` в `core.py`). Это автоматически
исключает мировой `index.m3u` и оставляет реально русскоязычные подборки.
